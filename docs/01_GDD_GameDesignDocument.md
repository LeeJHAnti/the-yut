## 1. 게임 개요 (Game Overview)

**타이틀:** The Yut  
**장르:** 턴제 멀티플레이어 보드 게임  
**플랫폼:** 웹 브라우저 (Godot 4 HTML5 Export)  
**플레이어 수:** 2~4인 (온라인 멀티플레이) / 1인 (AI 봇 포함)  
**테마:** 90년대 Game Boy 흑백 도트 아트  
**타겟 세션 시간:** 10~20분

### 1.1 핵심 컨셉

전통 윷놀이를 Game Boy 시대의 흑백 픽셀 아트로 재해석한 웹 기반 멀티플레이어 게임. 접속 후 2클릭 이내에 게임에 진입할 수 있는 극도로 간결한 UX와, 윷을 플릭해서 던지는 직관적인 조작, 말이 보드를 질주하는 임팩트 있는 애니메이션이 핵심 차별점이다.

### 1.2 디자인 필러 (Design Pillars)

1. **즉시 플레이**: 로딩 → 이름 입력 → 게임. 장벽 없는 진입.
2. **레트로 임팩트**: 흑백 도트지만 화면 흔들림, 속도선, 잔상 등으로 역동적인 연출.
3. **소셜 턴제**: 친구와 한 방에서 턴을 주고받는 간단하지만 전략적인 플레이.

## 2. 엔진 선정: Godot 4 (확정)

### 2.1 Godot vs Unity 비교 결과

| 기준 | Godot 4 | Unity |
|------|---------|-------|
| **웹 빌드 크기** | 2.7~6.4 MB (압축) | 8 MB+ (최소) |
| **2D 렌더링** | 네이티브 2D, 픽셀 퍼펙트 내장 | 3D 엔진 위 2D, 추가 설정 필요 |
| **WebSocket** | WebSocketMultiplayerPeer 내장 | 제한적, 서드파티 필요 |
| **라이선스** | MIT (완전 무료) | 매출 $200K 이상 시 유료 |
| **COOP/COEP 헤더** | 4.3+ 싱글스레드 빌드 시 불필요 | 해당 없음 |
| **픽셀 아트 지원** | 스냅 설정 1클릭 | 수동 설정 다수 필요 |
| **로딩 속도** | 빠름 (경량) | 느림 (무거운 런타임) |

**결론:** 흑백 도트 2D 게임 + 웹 배포 + 무료 라이선스 → **Godot 4 확정**. Unity는 이 프로젝트에서 오버엔지니어링.

### 2.2 Godot 4 사용 시 주의사항

- 웹 빌드는 싱글스레드 모드로 export (COOP/COEP 헤더 회피)
- WebGL 2.0 필수 — Safari 구버전 호환성 테스트 필요
- 모바일 브라우저 지원은 제한적, 데스크탑/태블릿 우선 타겟

## 3. 윷놀이 규칙 정의 (Game Rules Specification)

### 3.1 말판 (Board Layout)

```
            [출발/도착]
              (0)
             / | \
           /   |   \
        (19)  (1)  (20)
        /      |      \
      (18)    (2)    (21)
      /        |        \
   [좌상]    (3)    [우상]
    (5) -----(4)-----(?)
    |  \      |      /  |
   (6)  (23) (16) (24)  (?)
    |     \   |   /     |
   (7)    [중앙(22)]   (?)
    |     /   |   \     |
   (8)  (17) (11) (15)  (?)
    |  /      |      \  |
   [좌하]   (10)   [우하]
    (9) -----(?)-----(?)
      \        |        /
      (도착방향) (도착방향)
```

**노드 수:** 총 27개 노드 (0-26)
- 외곽 경로: 20개 노드
- 지름길 및 중앙 경로: 7개 추가 노드 (20~26)

**경로 종류:**
- **외곽 경로 (기본):** 0→1→2→...→19→0(도착) — 20개 노드의 순환
- **지름길 A (노드 5에서):** 5→20→22→23→15 (→외곽 계속)
- **지름길 B (노드 10에서):** 10→21→22→24→0(도착)
- **중앙 직행:** 22→25→26→0(도착) — 중앙을 정확히 밟았을 때만

### 3.2 윷 던지기 결과 및 확률

4개의 윷가락, 각각 앞면(평평)/뒷면(둥근) — 공정한 확률 가정:

| 결과 | 앞면 수 | 이동 칸 | 조합 수 | 확률 |
|------|---------|---------|---------|------|
| **모 (Mo)** | 0 (전부 뒤) | 5칸 | C(4,0) = 1 | 6.25% |
| **도 (Do)** | 1 | 1칸 | C(4,1) = 4 | 25.0% |
| **개 (Gae)** | 2 | 2칸 | C(4,2) = 6 | 37.5% |
| **걸 (Geol)** | 3 | 3칸 | C(4,3) = 4 | 25.0% |
| **윷 (Yut)** | 4 (전부 앞) | 4칸 | C(4,4) = 1 | 6.25% |

**특수 규칙:**
- **윷 또는 모** → 추가 턴 획득 (연속 가능, 제한 없음)
- **상대 말 잡기** → 추가 턴 획득

### 3.3 말 이동 규칙

- 플레이어당 **4개의 말** (전통 룰)
- 출발점에서 시계 반대 방향으로 이동
- **말 업기 (Stacking):** 같은 위치의 아군 말은 합체하여 함께 이동
- **잡기 (Capture):** 상대 말이 있는 칸에 도착 시 상대 말을 출발점으로 되돌림
- **업힌 말 잡기:** 업힌 그룹 전체가 잡힘 (고위험 고보상)
- **도착:** 말이 출발점을 정확히 통과하거나 도착하면 완주

### 3.3.1 엣지케이스 규칙 (추가 정의 필요)

- **업힌 그룹의 부분 잡기:** 업힌 말 그룹이 잡히면 그룹 전체가 출발점으로 돌아감
- **유효한 이동이 없는 경우:** 모든 말이 출발 전이고 결과가 "뒤도"(백도, 선택적 룰)인 경우 → 턴 스킵
- **연속 윷/모 시 이동 순서:** 모든 던지기를 완료한 후, 축적된 결과를 원하는 순서로 사용 가능
- **동일 칸 복수 경로 도달:** 지름길/외곽 모두 같은 노드에 도달 가능한 경우, 선택한 경로에 따라 다음 이동 방향이 결정됨
- **백도 (선택적 룰):** 도(Do) 결과 중 특수한 1가지 = -1칸 이동. 구현 여부는 Design Agent가 최종 확정

### 3.4 승리 조건

4개의 말 전부를 보드에서 완주시킨 첫 번째 플레이어가 승리.

### 3.5 턴 진행 흐름

```
턴 시작
  → 윷 던지기 (플릭 제스처)
  → 결과 표시 (도/개/걸/윷/모)
  → [윷 or 모?] → 예: 추가 던지기 저장, 계속 던지기
  → 모든 던지기 완료 후 → 이동할 말 선택
  → 말 이동 애니메이션
  → [상대 말 잡음?] → 예: 추가 턴
  → [지름길 분기점?] → 경로 선택 UI
  → 턴 종료, 다음 플레이어
```

## 4. 유저 플로우 (User Flow)

### 4.1 2클릭 진입 원칙

```
[웹페이지 접속]
    │
    ▼
[타이틀 화면: "The Yut"]
  - 이름 입력 필드 (기본값: 랜덤 닉네임)
  - [빠른 시작] 버튼  ← 클릭 1
  - [방 만들기] / [방 코드 입력]
    │
    ▼ (빠른 시작 클릭 시)
[대기 중인 방 자동 매칭 or 새 방 생성]
  - 방장이 [게임 시작] 클릭 ← 클릭 2
    │
    ▼
[게임 시작!]
```

### 4.2 화면 구성 — 3요소 원칙 (Three-Element Rule)

**핵심 원칙: 모든 화면에서 동시에 보이는 정보는 최대 3개.**
화면이 간결해야 플레이가 쉽다. 불필요한 정보는 숨기고, 현재 액션에 필요한 것만 표시.

#### 화면 1: 타이틀
1. 게임 로고 "THE YUT" + 이름 입력 (탭 수정)
2. **PLAY** 버튼 (빠른 시작 — 가장 큰 버튼)
3. **MORE** 토글 → 방 만들기 / 방 참가 (기본 숨김)

#### 화면 2: 대기실
1. **방 코드** (크게 표시, 공유용)
2. **플레이어 도트** (아이콘+짧은이름 — 리스트 아님, 시각적 도트)
3. **액션 버튼** (방장: 시작+봇추가 / 일반: 나가기)

#### 화면 3: 게임 플레이
1. **말판** (항상 표시 — 배경 역할, 말 직접 터치로 선택)
2. **상단 상태바** (한 줄: 현재 누구 턴인지만 표시)
3. **하단 액션 영역** (컨텍스트 전환 — 아래 중 하나만 표시):
   - 던지기 단계: 플릭 영역 + "Flick to throw!"
   - 말 선택 단계: "Tap a piece to move" (말판 위 말 직접 터치)
   - 경로 선택: 경로 버튼 2개 (OUTER / SHORTCUT)
   - 상대 턴: "{이름}'s turn..." (조작 불가)

#### 화면 4: 결과
1. **승자 텍스트** (큰 폰트, 바운스 애니메이션)
2. **PLAY AGAIN** 버튼
3. **EXIT** 버튼

## 5. 아트 디렉션 (Art Direction)

### 5.1 비주얼 스타일: Game Boy Monochrome

**색상 팔레트 (4색 제한 — Game Boy 정통):**
```
#0f380f  ██  가장 어두운 녹흑색 (배경, 외곽선)
#306230  ██  어두운 녹색 (그림자, 보조)
#8bac0f  ██  밝은 녹색 (주요 오브젝트)
#9bbc0f  ██  가장 밝은 녹색 (하이라이트)
```

> 참고: 사용자가 "흑백"을 요청했으나, Game Boy는 실제로 위 4색 녹색톤을 사용. 순수 흑백 옵션도 설정에서 제공:
> `#000000` / `#555555` / `#AAAAAA` / `#FFFFFF`

### 5.2 해상도 및 스케일링

- **기본 해상도:** 160×144 (Game Boy 원본)
- **렌더링:** 정수 배율 스케일링 (×2, ×3, ×4... 브라우저 크기에 맞춤)
- **픽셀 퍼펙트:** Godot의 `stretch_mode: viewport`, `stretch_aspect: keep`

### 5.3 필요 스프라이트 목록

| 카테고리 | 에셋 | 사이즈 | 비고 |
|----------|------|--------|------|
| **말판** | 보드 전체 | 128×128 px | 타일맵 또는 단일 스프라이트 |
| **말 (Player 1~4)** | 각 4종 | 8×8 px | 플레이어 구분용 패턴 차이 |
| **윷가락** | 앞/뒤 각 1 | 8×16 px | 던지기 애니메이션 프레임 |
| **윷 결과 텍스트** | 도/개/걸/윷/모 | 32×16 px | 스프라이트 폰트 |
| **UI 요소** | 버튼, 커서, 프레임 | 다양 | Game Boy 스타일 윈도우 |
| **이펙트** | 먼지, 별, 충격파 | 16×16 px | 8~12프레임 애니메이션 |
| **타이틀 로고** | "The Yut" | 80×32 px | 부팅 화면 포함 |

### 5.4 애니메이션 사양 (Enhanced)

모든 애니메이션은 `particle_effects.gd`의 경량 파티클 시스템 기반.
Game Boy 4색 팔레트 내에서 최대한 역동적인 연출.

#### 윷 던지기 애니메이션 (5단계)
1. **플릭 감지** → 4개 윷가락이 위로 날아감 (각각 다른 회전 속도)
2. **공중 회전** → 각 윷가락 독립적 spin + 포물선 궤적
3. **순차 착지** → 0.06초 간격으로 하나씩 바운스 착지, 먼지 파티클(DustPuff)
4. **결과 팝업** → 1.5배→0.9배→1.0배 펀치 스케일 바운스
5. **윷/모 특수** → 화면 플래시 + 줌 펀치(zoom_punch) + 별 폭발(StarBurst 12개) + "EXTRA TURN!" 뱃지 펄스

#### 말 이동 애니메이션 (홉 시퀀스)
1. **준비** → 스쿼시(1.2x, 0.7y) 웅크림
2. **도약** → 위로 10px 점프 + 스트레치(0.8x, 1.4y) + 속도선(SpeedLines) 파티클
3. **착지** → 랜딩 스쿼시(1.4x, 0.6y) + 먼지 파티클(DustPuff 4개) + 바운스 복원
4. **고스트 트레일** → 이전 위치에 반투명 잔상이 페이드아웃
5. **칸이 여러 개면** → 1~4번 반복 (홉-홉-홉)

#### 배치 애니메이션 (Home → Board)
- 위에서 드롭 + 바운스 착지 + 먼지 이펙트 + 임팩트 스쿼시

#### 잡기 애니메이션 (Capture)
1. **프리즈 프레임** → 0.06초 시간 정지(Engine.time_scale=0.05)
2. **충격파** → ImpactRing 확산 + StarBurst 6개
3. **튕겨남** → 잡힌 말이 회전하며 날아감(spin+fly) + 축소 + 페이드아웃
4. **카메라** → heavy_shake(0.3초)

#### 업기 애니메이션 (Stack)
- MergePulse 링 이펙트 + 말 위에 스택 도트 표시 (숫자 대신 시각적 도트)

#### 완주 애니메이션 (Finish)
1. 글로우 펄스 (1.8배 확대)
2. 상승 + 페이드아웃 + Sparkles 파티클 12개 + StarBurst 10개
3. 카메라 shake

## 7. 멀티플레이어 시스템

### 7.1 아키텍처

```
[Browser: Godot WASM]  ←WebSocket→  [Rust Server (Actix-Web)]
[Browser: Godot WASM]  ←WebSocket→         │
[Browser: Godot WASM]  ←WebSocket→    [Game Room Manager]
[Browser: Godot WASM]  ←WebSocket→         │
                                     [Game State Engine]
```

### 7.2 서버 책임 (Authoritative Server)

서버가 모든 게임 로직의 권위자(Authority):
- 윷 던지기 결과는 **서버에서 생성** (치트 방지)
- 말 이동 유효성은 **서버에서 검증**
- 클라이언트는 입력만 전송, 결과를 수신하여 렌더링

### 7.3 메시지 프로토콜 (JSON over WebSocket)

모든 메시지는 `{type, payload: {...}}` 구조를 사용:

```
// 클라이언트 → 서버
{ "type": "create_room", "payload": {} }
{ "type": "join_room", "payload": { "code": "A1B2" } }
{ "type": "quick_match", "payload": {} }
{ "type": "add_bot", "payload": { "difficulty": "normal" } }
{ "type": "change_name", "payload": { "name": "새이름" } }
{ "type": "start_game", "payload": {} }
{ "type": "throw_yut", "payload": { "gesture_power": 0.85 } }
{ "type": "select_piece", "payload": { "piece_id": 2 } }
{ "type": "select_path", "payload": { "path_choice": "shortcut" } }

// 서버 → 클라이언트 (브로드캐스트)
{ "type": "yut_result", "payload": { "result": "mo", "distance": 5, "extra_turn": true } }
{ "type": "piece_moved", "payload": { "player": 1, "piece_id": 2, "new_position": 10, "captured": [3] } }
{ "type": "game_state_sync", "payload": { "state": { ... full state ... } } }
{ "type": "player_joined", "payload": { "player": { "id": 2, "name": "플레이어2", "is_bot": false } } }
{ "type": "game_over", "payload": { "winner": 1 } }
{ "type": "bot_added", "payload": { "player_id": 3, "name": "봇", "difficulty": "normal" } }
```

### 7.4 방 시스템

- **방 코드:** 4자리 영숫자 (예: A1B2)
- **최대 인원:** 4명
- **방장:** 첫 생성자, 게임 시작 권한
- **자동 매칭:** "빠른 시작" 시 대기 중인 방에 자동 배정
- **AI 봇 지원:** 싱글플레이 또는 인원 부족 시 AI 봇 추가 가능 (난이도: easy/normal/hard)
- **연결 끊김:** 30초 재접속 대기 → 타임아웃 시 AI 대체 또는 턴 스킵
- **방 수명:** 게임 종료 후 5분 비활성 시 자동 삭제
